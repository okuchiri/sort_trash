# 抓取系统当前共识

更新时间: 2026-03-09

## 项目目标

目标是搭建一个固定场景下的桌面抓取与分类系统，硬件包括：

- 松灵 `NERO` 机械臂
- 智元 `OmniHand` 灵巧手
- Intel RealSense `D435` 深度相机

任务流程目标：

1. 固定相机观测桌面
2. 识别桌面物体
3. 得到物体位置
4. 将相机坐标转换为机械臂坐标
5. 机械臂移动到抓取位
6. 灵巧手抓取物体
7. 将物体放到指定投放点

应用风格类似：

- 垃圾分拣
- 回收分类

## 已确认的场景约束

- 相机型号：`Intel RealSense D435`
- 相机：固定安装
- 相机视角：暂未最终定死，但当前建议 `俯视优先`
- 抓取高度：第一版计划通过测试后 `写死`
- 第一版物体：
  - 空矿泉水瓶
  - 水杯
- 物体姿态：
  - 默认 `躺着`
  - 默认 `圆柱体`
  - `无把手`
- 任务模式：
  - 一次只抓 `一个物体`
- 投放点数量：
  - `2` 个
  - 矿泉水瓶一个
  - 水杯一个

## 当前技术判断

### 视觉路线

第一版优先建议：

- 用 `YOLO` 做检测
- 不把大模型 API 作为第一版主识别链路

原因：

- 输出结构化
- 速度更稳
- 更适合机械臂闭环控制

### 相机安装建议

第一版建议：

- `俯视安装`

原因：

- 标定简单
- 桌面坐标映射简单
- 与固定抓取高度更匹配
- 当前目标物体少，且一次只处理一个

### 抓取策略建议

第一版只做一个通用抓取流程：

1. 灵巧手张开
2. 机械臂到预抓取位
3. 机械臂下降到固定抓取高度
4. 灵巧手闭合
5. 机械臂抬起
6. 机械臂移动到投放点
7. 灵巧手张开放下

## 当前环境路线讨论结果

用户希望：

- 先在 `Windows` 上把能做的事情先做好
- 后续通过 `Mac SSH -> Ubuntu` 来运行系统

当前建议：

### 推荐主路线

- `Windows`：前期视觉开发与调试
- `Mac`：作为远程开发终端
- `Ubuntu`：作为最终运行机械臂 + 灵巧手的主控机

这条路线的理解方式是：

- Windows 不是最终执行端
- Mac 也不是最终执行端
- 真正负责运行抓取系统的是 Ubuntu
- Mac 只负责 SSH 到 Ubuntu 进行开发、启动、查看日志

### 为什么不建议 Windows 作为最终主控

根据当前已下载 SDK 文档：

- `pyAgxArm` 的 NERO 控制文档按 `Ubuntu + CAN` 使用
- `OmniHand-2025-SDK` 明确要求 `Ubuntu 22.04 x86_64`
- `pyAgxArm` 源码中 `Windows` 的 `CanComm` 分支不可直接用于当前这条官方控制链

因此：

- Windows 适合做前期视觉和调试
- Ubuntu 更适合做正式执行

### Mac 的角色建议

Mac 最适合承担：

- SSH 登录 Ubuntu
- 编辑代码
- 提交代码
- 启动/停止服务
- 查看日志

Mac 不建议承担：

- 直接作为最终机械臂 + 灵巧手控制主机

因为当前官方 SDK 的目标环境是 Ubuntu，而不是 macOS。

## 关于“Windows 继续做”还是“直接 Mac -> Ubuntu”

### 方案 A：先 Win，再转 Ubuntu

适合情况：

- 你希望先快速把 D435、YOLO、数据采集、标注、可视化跑起来
- 你现在手头最熟的是 Windows
- 你还没有把 Ubuntu 主控机完全准备好

优点：

- 上手快
- 调试界面方便
- 前期验证成本低

缺点：

- 后面仍然要做一次环境迁移
- 视觉代码和执行代码会分两套环境

### 方案 B：直接 Mac -> Ubuntu

适合情况：

- 你愿意一开始就统一环境
- 你准备把最终系统长期稳定跑在 Ubuntu
- 你不想后面再做迁移

优点：

- 环境统一
- 后期最省事
- 更接近最终部署状态

缺点：

- 前期上手没有 Windows 直观
- 某些厂家工具在 Windows 上更方便调试

### 当前综合建议

最稳的路线是：

- `Windows` 只保留给厂家工具和前期视觉验证
- 从现在开始，真正的项目代码尽量以 `Ubuntu` 为目标环境
- 你平时主要通过 `Mac SSH -> Ubuntu` 工作

一句话总结：

- 可以先用 Win
- 但不要把 Win 变成主战场
- 主战场应该尽快切到 `Mac SSH -> Ubuntu`

## 当前硬件接口判断

### NERO 机械臂

按当前文档判断，大概率需要：

- `USB 转 CAN` 模块

### OmniHand 灵巧手

按当前文档判断，需要：

- `ZLG USBCANFD` 系列模块

推荐型号：

- `USBCANFD-100U-mini`
- `USBCANFD-100U`
- `USBCANFD-200U`

注意：

- 目前只能确认“系统需要这些模块”
- 不能从现有文件直接确认“用户手上已经有这些模块”

## 接下来建议的工作顺序

### 第一阶段

- 确认 Ubuntu 主控机可用
- 确认机械臂和灵巧手所需 CAN / CANFD 硬件模块
- 确认 D435 固定安装方案

### 第二阶段

- 在 Windows 或 Ubuntu 上完成 D435 取流
- 完成物体检测最小验证
- 完成桌面坐标到机械臂坐标的标定方案

### 第三阶段

- 在 Ubuntu 上分别跑通：
  - NERO 单独控制
  - OmniHand 单独控制

### 第四阶段

- 联通视觉输出与抓取执行
- 完成两类物体的抓取与投放
